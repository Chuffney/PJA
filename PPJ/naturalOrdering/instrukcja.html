Sortowanie naturalne

Popularną pułapką w domyślnym sposobie sortowania dostępnym w większości języków programowania jest to, jak są traktowane liczby w tekście. Mówiąc prościej, gdy dany string zawiera cyfry, wynikowe porządki sortowania mogą być zaskakujące. Przykładowo:

#include <algorithm>
#include <string>
#include <vector>

#include <fmt/ranges.h>

auto main() -> int {
    auto fileNames = std::vector<std::string>{
        "lecture 1", "lecture 2", "lecture 3",
        "lecture 10", "lecture 11", "lecture 12",
        "lecture 35", "lecture 39", "lecture 92",
        "lecture 101", "lecture 111", "lecture 133",
        "lecture 159", "lecture 212", "lecture 221"
    };

    std::ranges::sort(fileNames);

    fmt::println("{}", fmt::join(fileNames, "\n"));
}

Niewątpliwie jest to zaskakujący wynik, zwłaszcza jeżeli ktoś nie był świadomy tego zachowania wcześniej.

Mimo wszystko takie zachowanie jest uzasadnione – w końcu tekst jest porównywany leksykograficznie, czyli słownikowo. Skoro "aaa" powinno być przed "z", to czemu 111 nie miałoby być przed 9, prawda?

Z drugiej strony nie jest to zachowanie, którego najczęściej pożądamy, zwłaszcza gdy – przykładowo – porządkujemy pliki na komputerze.

Implementacja

Zmodyfikuj listing kodu zaprezentowany wcześniej tak, aby algorytm sortowania posortował elementy wektora fileNames zgodnie z sortowaniem naturalnym.

Ulepszenie implementacji

Stworzony komparator (lub projekcja) jest najpewniej bardzo obszerny. Ulepsz implementację tworząc przestrzeń nazw pjc, w której umieścisz funkcję naturalOrder(), która zwróci stworzony komparator. Korzystając z nowej podprzestrzeni nazw wydziel pomocnicze funkcje / lambdy użyte w implementacji naturalOrder().
